## 顺序查找和折半查找

### 顺序查找

* 优点：对数据元素的存储没有要求
* 缺点：平均查找长度较长，效率低
* 有序时可以降低查找失败的平均查找长度

### 折半查找（二分查找）

```py
def binary_search(arr: list, target):
    lo, hi = 0, len(arr) - 1
    while lo <= hi:
        mid = (lo + hi) // 2
        if arr[mid] == target:
            return mid
        elif arr[mid] > target:
            lo = mid + 1
        else:
            hi = mid - 1
    return -1
```

* 判定树
* 优点：速度快·
* 缺点：要求线性表具有随机存储的特性，仅适合顺序存储结构，不适合于链式存储

### 分块查找（索引顺序查找）

* 块内元素可以无序，但块之间是有序的
* 顺序查找或折半查找所在的块，然后在块内顺序查找
* 若有n个元素，构造成索引顺序查找，则根据均值不等式，每块sqrt(n)个元素最佳
* 若块间使用折半查找，则是分得越小越好
